# Прогрев аккаунтов — выжимка для AF-scenarios

> Источники: `прогрев+выкладка (1).docx`, `Мануал по прогреву аккаунтов.docx`  
> Используется в `variables.yaml` → `warmup_profiles` и ограничениях orchestrator.

## Общие принципы (все площадки)

- **Не спешим** — резкая активность после регистрации подозрительна; нужна отлёжка.
- **Поведение как у человека** — сначала просмотр, потом лайки, затем комменты и подписки.
- **Тематический поиск** — алгоритм учится на том, что ищем и смотрим.
- **Критерий готовности** — лента рекомендаций показывает тематический контент (TikTok/IG: 6–7 из 10 видео).

## TikTok — прогрев вокруг публикации (для daily-сценариев)

Используется в шагах `warmup_feed` с профилем `tiktok_daily`:

| Фаза | Действие | Длительность |
|------|----------|--------------|
| **pre_publish** | Лента «Рекомендации», только просмотр + редкие лайки | 5–10 мин |
| **post_publish** | То же после поста | 5–10 мин |

### Лимиты активности (поиск по ключам, отдельная фаза)

- 10–15 мин на один ключ
- 3–5 лайков максимум за сессию по ключу
- Можно лайкать комментарии, читать их
- Вариации ключа: «футбол edit», «футбол момент» и т.п.

### Запреты (enforce в orchestrator)

| Запрет | Причина |
|--------|---------|
| Много **сохранений** видео | Теневой бан |
| Переключение аккаунтов за 10–15 мин | Антиспам |
| Массовые однотипные лайки/комменты | Теневой бан |
| Одинаковый набор хэштегов на каждый пост | Шаблон/бот |

## Instagram

База как TikTok + при прогреве по поиску: лайк комментов, свои комменты, копирование ссылки на видео (без вставки).

## YouTube

**Прогрев после публикации**, не до. ~30 мин по тематическим ключам после выкладки.

## Facebook

Упор на комментарии к постам из ленты, подписки, обратная связь. Осторожно с **banwords**.

## Публикация контента (раздел 7 мануала)

- **Хэштеги:** 3–7 релевантных, ротировать, не копировать шаблон
- **Описание:** цепляющая первая строка, без keyword-stuffing
- **Формат TikTok/Reels:** 9:16, 1080×1920, хук в первые 1–3 сек
- **Частота:** стабильно ~1 в день лучше, чем пачкой
- **Прайм-тайм TikTok:** вечер 19:00–23:00 (ориентир; смотреть статистику аккаунта)

## Маппинг в variables.yaml

```yaml
warmup_profiles:
  tiktok_daily:
    pre_publish:
      mode: recommendations_feed
      duration_sec: [300, 600]
      likes_max: [0, 2]
      saves: forbidden
    post_publish:
      duration_sec: [300, 600]
      likes_max: [1, 3]
      saves: forbidden
```

Полный пример: `examples/saxonov-tiktok-daily/variables.yaml`.
